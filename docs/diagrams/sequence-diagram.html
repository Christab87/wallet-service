e-Wallet: Complete Flow

Sequence Diagram - Mint → Swap → Melt

Overview

This diagram shows the complete interaction flow between all components of the Cashu e-wallet system:

  • User - Frontend requesting payments
  • Flask App - REST API (Port 8000)
  • Wallet Service - State management
  • Cashu Client - Protocol handler
  • Blind Signing - Cryptography layer
  • Mock Mint - Payment server (Port 5001)
  • Storage Service - Encrypted proof storage

1️⃣ MINT PHASE

Generate 100 sats in 7 proofs

  • Health check
  • Request quote
  • Blind amounts
  • Blind sign
  • Unblind & store

2️⃣ SWAP PHASE

Privacy - swap 3 proofs for 7 new ones

  • Get keys
  • Select proofs
  • Blind new outputs
  • Server swaps
  • Unblind & replace

3️⃣ MELT PHASE

Redeem 10 sats for Lightning

  • Request quote
  • Select proofs
  • Submit to server
  • Verify proofs
  • Return invoice
